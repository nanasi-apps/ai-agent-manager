1. メインダッシュボード（俯瞰画面）
すべてのエージェントの稼働状況と、リソースの状態をリアルタイムに把握する画面です。
 * エージェント・ステータス一覧: 起動中の Agent.exe、使用中のモデル、現在のタスク進行度をカード形式で表示。
 * リソースモニター: 各 Git Worktree のディスク使用量、CPU/メモリ使用率の可視化。
 * 共有DBログ: エージェント間での「意図（Intent）」の予約状況（例：「Agent A が Auth.ts をロック中」）のタイムライン表示。
2. エージェント画面
個別エージェントとの対話および、詳細な実行制御を行う画面です。
 * stdio をパースし、フロントエンドでチャット型に表示。
 * モデルホットスワップ: 実行中に LLM モデル（GPT-5 ⇄ Claude Sonnet 4.5 等）を切り替える UI。
 * チェックポイント制御: 会話履歴や思考状態のスナップショット保存・復元。
 * MCP ツール連携表示: エージェントが現在どの MCP Tool を呼び出しているか、その引数と戻り値をリアルタイム表示。
 * CLIツール結合：Gemini-CLIや、Claude-Code、CodexなどのCLIツールをエージェントとして使える様にする。
 * API結合：OpenAI APIやClaude API, Anthropic APIなどを直接叩くことも可能。
3. Git Worktree ＆ 並行実行管理画面
「ファイルが被る」問題を物理的・論理的に解決するための管理画面です。
 * Worktree マネージャー: 並行実行用の Worktree 作成、削除、パス管理。
 * マイクロコミット・ログ: エージェントが行った細粒度なコミット履歴の視覚化。
 * コンフリクト・シミュレーター: マージ前に競合を予測し、エージェントへのフィードバック状態（Rebase待ちなど）を表示。
 * pnpm/キャッシュ管理: 共有 node_modules やビルドキャッシュの最適化設定。
4. ルール ＆ MCP 設定画面
エージェントの「脳」と「手足（ツール）」を定義・同期する画面です。
 * プロンプト・エディタ: ルール（システムプロンプト）の編集、バージョン管理、Diff 表示。
 * クラウド同期設定: Supabase 等を通じた、複数端末間でのルール・ログイン情報の同期。
 * MCP ハブ設定: * 接続中の MCP サーバー（Agent.exe, DB Tool, etc.）の一覧。
   * 利用可能な Resources (URI) と Tools の定義確認。
5. 競合解決・オーケストレーション画面
論理的な矛盾が発生した際に、人間が介入するためのインターフェースです。
 * インパクト・アナライザー: 依存関係グラフに基づき、複数のタスクが同じロジックを破壊する可能性を警告。
 * 3-way Merge UI: エージェント同士で解決できなかったコードの競合を、ユーザーが最終判断するためのエディタ。
 * 意思決定チャット: エージェントから「どちらの実装が良いか？」という相談をプッシュ通知で受け取り、指示を出す。
システムの全体構造（イメージ）
ダッシュボードが中心となり、MCP を介してすべてを繋ぎます。
